# LumoRPA 优化与补充开发路线图

> 生成于 2026-05-29。基于对 8 个 crate + Tauri 桌面端(~21.6k 行)的全量子系统审计 + 关键结论的人工 grep 复核。
> 本文既是分析报告,也是执行清单。状态图例:`[ ]` 待办 · `[~]` 进行中 · `[x]` 已完成。

## 总体判断

架构扎实的 M1/M2 阶段本地优先 RPA 运行时。错误模型统一、能力沙箱/网络开关基本健全、录制器与"影刀式编辑器"均已真正实现(README 的 placeholder 标签过时)、actions 多达 91 个(文档承诺 50)。

**但**:最新 P0 提交的招牌功能(AI 痕迹闭环 + 多模态抽取)因"最后一公里"未接线而**实际不运行**,且存在数个真实安全边界漏洞。

## 🟠 P1 — 健壮性 / 可用性 / 工程化

- [x] **P1-1 VM 无取消/超时** ✅ 已修复
  - 新增 `lumo_core::CancelToken`(基于 `watch`,`cancel()`/`cancelled()`/`is_cancelled()`);`FlowVm` 增 `with_cancel`/`with_step_timeout`;`StepCtx` 携带 cancel/timeout 并在 `fork` 中传递。`execute_step` 每步前检查取消、用 `select!`(biased) 在执行中并发监听取消/`tokio::time::sleep` 超时;取消→`ExecError::Cancelled`、超时→`ExecError::Timeout`。run 结束按结果落 `cancelled`/`failed`/`ok` 状态。`lumo run` 接 Ctrl-C→取消(退出码 130)。4 单测覆盖运行中取消/运行前取消/超时/无限制正常。
- [x] **P1-2 浏览器进程失败时不回收**(`browser.rs:157`)✅ 已修复
  - 新增 `lumo_core::RunTeardown` 钩子 + `ActionRegistry::register_teardown`;`FlowVm::run` 在流程结束(成功/失败均执行)按 `run_id` 调用全部钩子。`browser.rs` 注册 `BrowserTeardown`,`Session` 改 `tokio::sync::Mutex<Browser>` + 可中止的 handler 任务,`teardown_session` 做 `close().await`→(失败则 `kill()`)→`wait()` 回收子进程。新增 lumo-core 端到端测试(成功/失败两路均触发 teardown)+ lumo-actions 真 Chrome 集成测试(`#[ignore]`,本地已验证回收)。
- [x] **P1-3 加密 vault 仅有表无实现** ✅ 已修复
  - 新增 `lumo-storage::vault` 模块:`VaultIdentity`(age X25519 身份,generate/load/save-0600/recipient)+ `encrypt`/`decrypt`(age 一次性 API)+ `Vault<'a>` 门面(put/get/delete,JSON 对象整体加密)+ `list`(仅元数据,永不解密)+ `get_field`(运行时单字段解密)。`Repo` 加 `vault_put/get/list/delete` 裸行方法(`vault_items` 表已在 baseline DDL)。
  - `lumo-core`:`StepCtx`/`FlowVm` 经 `with_vault` 携带不透明 `Arc<VaultIdentity>`(core 不直依赖 age);`${{ vault.* }}` 解析重构为 `VaultResolver`——**env 优先**(`LUMO_VAULT_*`,向后兼容/CI 覆盖)、**加密 store 回退**(repo+identity 均在时调 `get_field`)、都无则报错;身份缺失则优雅降级为纯 env。模板层 `preprocess_vault` 用 `{% raw %}` 将 `{{ vault.* }}` 真正字面量化(秘密绝不进入实时模板引擎/快照/日志)。
  - CLI 新增 `lumo vault`(init/add/get/list/rm/path):密文仅经隐藏 prompt(rpassword)或 `--stdin`,**绝不**上 argv;`get` 默认掩码、`--reveal` 出明文;`list` 永不出明文。run/serve/hotkey/mcp 运行器从 `$LUMO_HOME/age-identity.txt`(可经 `LUMO_VAULT_IDENTITY` 覆盖)加载身份注入 VM;子流程运行器(skills)保持纯 env。
  - 依赖:`age = "0.11"`(纯 Rust,无 C,利好交叉编译)+ `rpassword = "7"`。测试:lumo-storage 加解密/CRUD/门面往返 13 例 + lumo-core 解析优先级 6 例 + VM 端到端 + lumo-dsl 占位符边界 + lumo-cli 纯逻辑单测。
- [x] **P1-4 AI hook 调用不进成本账本** ✅ 已修复
  - 新增 `lumo_core::AiCallUsage` + `AiHookProvider::take_usage`(默认空实现 → 旧 impl/测试 stub 不破);`AiHooks` 内置 `Mutex<Vec<AiCallUsage>>` 累加器,5 个 helper(heal_selector/extract_visual/decide/vision_locate/diagnose)计时 + 算价后入账。成本表 `cost_micro` 抽到 `lumo-ai::cost`,与 `ai.chat` 动作共用(消除重复)。

- [x] **P1-5 多模态无图片大小限制 + provider 无 429 重试** ✅ 已修复
