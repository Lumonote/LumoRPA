# LumoRPA 优化与补充开发路线图

> 生成于 2026-05-29。基于对 8 个 crate + Tauri 桌面端(~21.6k 行)的全量子系统审计 + 关键结论的人工 grep 复核。
> 本文既是分析报告,也是执行清单。状态图例:`[ ]` 待办 · `[~]` 进行中 · `[x]` 已完成。

## 总体判断

架构扎实的 M1/M2 阶段本地优先 RPA 运行时。错误模型统一、能力沙箱/网络开关基本健全、录制器与"影刀式编辑器"均已真正实现(README 的 placeholder 标签过时)、actions 多达 91 个(文档承诺 50)。

**但**:最新 P0 提交的招牌功能(AI 痕迹闭环 + 多模态抽取)因"最后一公里"未接线而**实际不运行**,且存在数个真实安全边界漏洞。

  - `with_ai_provider()`(`crates/lumo-core/src/vm.rs:82`)**零调用者**;`AiHooks`(`crates/lumo-ai/src/helpers.rs:336`)从未构造;9 处生产 `FlowVm::new` 都没链 `.with_ai_provider(...)`。
  - `effective_ai_mode`(`vm.rs:1043`)在 `ctx.ai_provider().is_none()` 时直接返回 Off ⇒ heal_selector / extract_visual / decide / vision_locate 全部短路,一次都不触发。
  - **修复**:在每个 runner 的 `FlowVm::new` 站点按 `flow.metadata.ai.enabled` 链 `.with_ai_provider(Arc::new(AiHooks::new(router, budget)))`,加端到端集成测试。
- [x] **P0-2 能力沙箱路径不规范化(`..`/软链接逃逸)**(已人工核实)✅ 已修复

  - **修复**:flow 读写限制到 flows 根(复用 `read_artifact_blob` 的规范化校验);设置严格 CSP。

## 🟠 P1 — 健壮性 / 可用性 / 工程化

- [ ] **P1-1 VM 无取消/超时**(已核实:`vm.rs` 无 `CancellationToken`/`timeout`)— 接 `CancellationToken` + 每步 `tokio::time::timeout`,新增 `cancelled` run 状态。
- [x] **P1-2 浏览器进程失败时不回收**(`browser.rs:157`)✅ 已修复
  - 新增 `lumo_core::RunTeardown` 钩子 + `ActionRegistry::register_teardown`;`FlowVm::run` 在流程结束(成功/失败均执行)按 `run_id` 调用全部钩子。`browser.rs` 注册 `BrowserTeardown`,`Session` 改 `tokio::sync::Mutex<Browser>` + 可中止的 handler 任务,`teardown_session` 做 `close().await`→(失败则 `kill()`)→`wait()` 回收子进程。新增 lumo-core 端到端测试(成功/失败两路均触发 teardown)+ lumo-actions 真 Chrome 集成测试(`#[ignore]`,本地已验证回收)。
- [ ] **P1-3 加密 vault 仅有表无实现**(已核实:`vault_items`/`age_ciphertext` 仅在 `schema.rs:70/72`)— 实现 age 加解密 repo 方法 + `lumo vault` 命令 + `{{vault.*}}` 走 vault 解析。
- [ ] **P1-4 AI hook 调用不进成本账本**(已核实:`record_ai_call` 仅 `action.rs:153`)— 5 个 helper 补 `record_ai_call`;`_ai` 痕迹补 tokens/latency。
- [x] **P1-5 多模态无图片大小限制 + provider 无 429 重试** ✅ 已修复
  - `helpers.rs` 加 `MAX_IMAGE_BYTES=4MiB` 守卫(`extract_visual`/`vision_locate` 编码前校验);vision 解析失败改 `tracing::warn`(含截断原文)再回退默认值;`provider.rs` 加 `send_with_retry`(429/5xx + 瞬时网络错误指数退避,最多 3 次,4xx/解码错误保持终态)。wiremock 测试 4 例(重试成功/401 不重试/超大图拒绝)。
- [x] **P1-6 `lumo run` 失败仍退出 0**(`crates/lumo-cli/src/cmd/run.rs`)✅ — 失败时 `process::exit(1)`。
- [x] **P1-7 存储无 schema 迁移框架 + 无 busy_timeout** ✅ 已修复
  - `repo.rs` 加 `LATEST_USER_VERSION` + `run_migrations`(按 `PRAGMA user_version` 有序应用,已现版本短路);`init_connection` 每连接显式设 `busy_timeout(5s)`/`foreign_keys=ON`/`journal_mode=WAL`/`synchronous=NORMAL`;旧 `step_runs` 重建为 v1、baseline DDL 为 v2(修正了 DDL-先于-重建的列缺失 bug)。新增 5 例迁移测试(全新库/幂等重开/旧库原地迁移/PRAGMA 生效)。
- [ ] **P1-8 测试覆盖不足** — `lumo-actions`(91 actions)补集成测试;CI 接 `cargo-llvm-cov` 覆盖率下限。
- [ ] **P1-9 CI 缺供应链/覆盖率/MSRV/信创验证** — 加 `cargo-deny` + coverage + MSRV(1.83)+ 交叉 `cargo check`(aarch64/loongarch64)。
